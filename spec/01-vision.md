# Curiosity OS

## Document 1 — Vision & Product Philosophy

**Version:** v1.0

**Status:** Foundation

---

# Vision

Curiosity OS is a personal operating system for curiosity.

It is designed to become an external extension of my thinking—not another productivity tool, note-taking application or project management board.

Its purpose is to make curiosity easy to:

* Capture
* Pause
* Resume
* Connect
* Evolve
* Build

The system should preserve how my understanding changes over time rather than simply storing information.

---

# The Problem

I naturally think in many parallel threads.

At any given time I might be interested in:

* Product ideas
* AI
* Psychology
* Personal productivity
* Startups
* System design
* Business ideas
* Learning
* Research
* Life decisions

Most of these ideas never disappear because they are bad.

They disappear because:

* my attention shifts
* I become busy
* I temporarily lose interest
* I forget where I left off
* rebuilding context becomes expensive

Months later, I often remember an idea and think:

> "I was excited about this once. Why?"

That context is usually gone.

Curiosity OS exists to preserve that context.

---

# Product Philosophy

Curiosity OS is not a place where thinking happens.

Thinking happens in many places:

* ChatGPT
* Claude
* GitHub Copilot
* VS Code
* Voice notes
* Walking
* Reading
* Watching videos
* Conversations

Curiosity OS is where thinking becomes durable.

It becomes the long-term memory of those explorations.

---

# Core Principles

## Curiosity over Productivity

The system should encourage exploration.

It should never feel like work.

It should never pressure me into completing tasks.

Unlike productivity tools, success is not measured by completion.

Success is measured by sustained curiosity.

---

## Resume over Search

Finding information is easy.

Rebuilding context is hard.

The primary goal of the system is to minimize the cost of resuming an idea after hours, weeks or months.

Every design decision should optimize for fast resumption.

---

## Knowledge over Notes

The goal is not to accumulate Markdown files.

The goal is to accumulate understanding.

Markdown is only the storage format.

Knowledge is the actual product.

---

## Events over Snapshots

Knowledge changes.

The system should preserve:

* discoveries
* decisions
* mistakes
* invalidated assumptions
* dead ends
* breakthroughs

The evolution of thinking is often more valuable than the current state.

---

## Derived over Manual

Whenever possible, the system should infer information automatically.

Examples:

* Momentum
* Interest
* Confidence
* Activity
* Last Active
* Stage

These should rarely require manual editing.

---

## Portals over Integration

Curiosity OS should not replace specialized tools.

Instead it should coordinate them.

Examples:

* ChatGPT explores.
* Claude researches.
* GitHub Copilot builds.
* VS Code develops.
* GitHub stores code.

Curiosity OS remembers.

---

## Low Friction over Zero Friction

The goal is not complete automation.

A small amount of intentional review before publishing knowledge is valuable.

Reflection is part of thinking.

Automatic dumping creates noise.

Publishing should require almost no effort, but still allow the user to decide what becomes permanent knowledge.

---

## One Source of Truth

The Git repository is the canonical knowledge store.

Everything else is temporary.

External portals may change over time.

The repository should remain stable.

---

## Portable by Design

Everything important should live as plain text.

Prefer:

* Markdown
* JSON
* Images
* Git

Avoid vendor lock-in.

The repository should remain usable for years regardless of which AI tools exist.

---

# What Curiosity OS Is NOT

It is NOT:

* Jira
* Trello
* Linear
* Asana
* Notion
* Apple Notes
* A task manager
* A knowledge dump
* A second brain
* A documentation tool

Although it shares ideas with some of these products, its purpose is fundamentally different.

---

# The Emotional Experience

This is perhaps the most important requirement.

Opening Curiosity OS should feel like walking into a personal library of fascinating unfinished worlds.

It should create emotions like:

* Wonder
* Curiosity
* Excitement
* Discovery
* Momentum

It should never create emotions like:

* Guilt
* Pressure
* Obligation
* Backlog anxiety

Every interaction should reinforce curiosity rather than productivity.

---

# The "Magical Place"

Curiosity OS should feel magical.

Not through excessive animations or visual effects.

But through helping me rediscover forgotten excitement.

Examples:

Instead of showing:

* 43 ideas

Show:

> "This idea became interesting again."

Instead of:

* Recently Modified

Show:

> "You discovered something important here."

Instead of:

* Open Tasks

Show:

> "You left an interesting question unanswered."

The interface should feel alive.

---

# Mental Model

The system should be thought of as an Operating System rather than an application.

Its responsibilities are:

* Maintain memory
* Organize knowledge
* Coordinate portals
* Preserve history
* Resume thinking
* Connect ideas

It does not replace the applications used for exploration.

---

# Design Goals

The UI should be:

* Mobile-first
* Calm
* Lightweight
* Elegant
* Minimal
* Beautiful
* Highly readable

Avoid:

* Enterprise dashboards
* Dense tables
* Large spreadsheets
* Kanban boards
* Ticketing interfaces
* Project management aesthetics

The experience should feel closer to exploring a museum or library than managing work.

---

# Success Metrics

The system is successful if:

* I capture ideas without hesitation.
* I frequently return to previously abandoned ideas.
* Resuming an idea takes less than one minute.
* I naturally discover connections between different worlds.
* My curiosity compounds over years rather than resetting.

---

# Long-Term Vision

Over time, Curiosity OS becomes a version control system for thinking.

Every idea evolves.

Every exploration leaves a trace.

Every breakthrough builds on previous curiosity.

Rather than asking:

> "What have I accomplished?"

The system helps answer:

* What has fascinated me?
* How has my thinking evolved?
* What ideas keep resurfacing?
* Which curiosities are converging into something bigger?

The ultimate goal is not to maximize productivity.

The goal is to preserve and compound curiosity over a lifetime.

---

# Guiding Question

Whenever implementing a new feature, always ask:

> **Does this make it easier to capture, resume, connect or evolve curiosity?**

If the answer is no, the feature probably does not belong in Curiosity OS.
